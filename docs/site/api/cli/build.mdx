# `@zenbujs/cli/build`

The build-config types and helpers, imported from your `zenbu.build.ts`.

```typescript
import {
  defineBuildConfig,
  stripIfDisabled,
  dropFiles,
  type BuildConfig,
  type Transform,
  type Hook,
  type Adapter,
} from "@zenbujs/cli/build"
```

## `defineBuildConfig(config)`

```typescript
function defineBuildConfig(config: BuildConfig): BuildConfig
```

Identity function with type validation. See [Define build config](/build/define-build-config) for the field-by-field walkthrough.

## `stripIfDisabled(flags)`

```typescript
function stripIfDisabled(flags: Record<string, boolean>): Transform
```

Removes lines between `// @zenbu:strip-if !FLAG` and `// @zenbu:end` markers when `FLAG` is false in the passed map. See [Transforms](/build/transforms#built-in%3A-stripifdisabled).

## `dropFiles(pattern)`

```typescript
function dropFiles(pattern: RegExp): Transform
```

Drops every file whose path matches the regex. See [Transforms](/build/transforms#built-in%3A-dropfiles).

## Types

```typescript
interface BuildConfig {
  source?: string                       // default "."
  out?: string                          // default ".zenbu/source"
  include: string[]
  ignore?: string[]
  transforms?: Transform[]
  adapter?: Adapter                     // default liveSource()
  hooks?: Hook[]
}

interface TransformInput {
  path: string                          // file path relative to source root
  code: string                          // file contents as UTF-8 string
}

interface TransformOutput {
  code?: string
  drop?: boolean
}

type Transform = (file: TransformInput) => TransformOutput | null | undefined | void
```

```typescript
interface Hook {
  name: string
  resolveSourceConfig?(ctx: SourceCtx): SourceCtx | void
  transformFile?(file: TransformInput): TransformOutput | void
  finalizeSource?(ctx: SourceCtx): Promise<void> | void
  shapeBundle?(ctx: BundleCtx): Promise<void> | void
  postBundle?(ctx: BundleCtx): Promise<void> | void
}
```

## `Adapter` interface

For users writing their own adapters:

```typescript
interface Adapter {
  name: string

  resolveSourceConfig?(ctx: SourceCtx): SourceCtx | void
  transformFile?(file: TransformInput): TransformOutput | void
  finalizeSource?(ctx: SourceCtx): Promise<void> | void

  shapeBundle?(ctx: BundleCtx): Promise<void> | void
  extendElectronBuilder?(base: ElectronBuilderConfig, ctx: BundleCtx): ElectronBuilderConfig

  postBundle?(ctx: BundleCtx): Promise<void> | void

  runtime: {
    mode: "live" | "frozen"
    mainEntry: string                   // path to the gate file
    needsToolchain?: { bun: boolean; pnpm: boolean; git: boolean }
    needsMirrorPull?: boolean
    needsAppsDirProvision?: boolean
  }
}
```

The framework ships two adapters built on this interface:

- `liveSource()` from `@zenbujs/cli/adapters/live-source` — see [Live-source adapter](/build/live-source-adapter).
- `frozenSource()` from `@zenbujs/cli/adapters/frozen-source` — see [Frozen-source adapter](/build/frozen-source-adapter).

Custom adapters are an explicit extension point. If you write one and it's broadly useful, send a PR.

## What's *not* public

- `resolveBuildConfig` — internal. The result of `defineBuildConfig` is the public input shape.
- `MirrorConfig`, `BundleConfig` — replaced by adapter-specific options. The old top-level `mirror:` and `bundle:` fields are deprecated; use `adapter: liveSource({...})` instead.
