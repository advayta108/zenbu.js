# `zen`

The `zen` binary is shipped by `@zenbujs/cli`. Run it via the project's `package.json` scripts:

```bash
nr zen <command>
nr zen --help
```

Or globally:

```bash
nr -g add @zenbujs/cli
zen <command>
```

## Commands at a glance

| Command | Section |
| --- | --- |
| `zen` (no args) | Open the app |
| `zen dev` | Run app in dev mode |
| `zen init` | Scaffold a new plugin |
| `zen install` | Run `pnpm install` for an app |
| `zen setup` | Run a plugin's `setup.ts` |
| `zen doctor` | Re-run the framework's setup checks |
| `zen link` | Regenerate registry types |
| `zen db` | Database management commands |
| `zen kyju` | Inspect on-disk database |
| `zen exec` | Run TS with `rpc`/`events` pre-opened |
| `zen profile` | CPU profile / heap snapshot the kernel |
| `zen plugin` | List, toggle, remove plugins |
| `zen runtime` | Manage cached Electron runtimes |
| `zen build:source` | Run the TS-to-TS source build |
| `zen build:desktop` | Bundle the source into a binary |
| `zen publish:source` | Push staged source to mirror repo |
| `zen publish:desktop` | Upload binary to release host |
| `zen advice` | Inspect active advice/content scripts (planned) |

## `zen` (no args) — open

Opens the current directory's app in a managed Electron instance. Equivalent to `zen .`.

```bash
zen [path] [--blocking] [--verbose]
```

| Flag | Meaning |
| --- | --- |
| `[path]` | Project directory. Defaults to `.`. |
| `--blocking` | Keep zen attached to the Electron process — useful when piping logs. |
| `--verbose` | Print boot details. |

## `zen dev`

Run the app in development mode, with HMR and tsx loaded. Most apps just use `nr dev` (which calls `electron . --project=.` internally).

```bash
zen dev
```

## `zen init <name>`

Scaffold a new plugin in the current directory:

```bash
zen init my-plugin --dir ./plugins
```

| Flag | Meaning |
| --- | --- |
| `--dir <path>` | Where to create the plugin (defaults to `.`) |

## `zen install [path]`

Install npm dependencies for an app, using the bundled pnpm:

```bash
zen install                           # current directory
zen install /path/to/app
zen install /path/to/app --force      # full reinstall
```

## `zen setup`

Run a plugin's `setup.ts` script. Used after manual cloning:

```bash
zen setup --dir ~/.zenbu/plugins/my-plugin
```

## `zen doctor`

Re-run the framework's setup checks (toolchain availability, env paths, etc.). Useful when the framework misbehaves on a new machine:

```bash
zen doctor
```

## `zen link`

Regenerate the per-app registry types in `<app>/types/`:

```bash
zen link
zen link --types-config ./zenbu-types.config.json --registry ./types
```

Most users never run this manually — the dev server runs it on boot and on manifest changes.

## `zen db`

Database management commands.

```bash
zen db list                          # list configured DB paths
zen db add /path/to/db
zen db default /path/to/db
zen db remove /path/to/db

zen db generate --name add_user_email   # generate a migration for the current schema
zen db generate --schema src/schema.ts --migrations db/migrations --name foo
```

The `generate` subcommand is what you run after editing your schema. See [Migrations](/db/migrations).

## `zen kyju`

Inspect on-disk database state for the configured (or specified) DB:

```bash
zen kyju db root                     # print root JSON
zen kyju db collections              # list collections
zen kyju db collection <id>          # print a collection's items
```

Read-only. Useful for debugging when you suspect data corruption.

## `zen exec`

Run a TypeScript file with `rpc` and `events` pre-opened against a running app:

```bash
zen exec -e 'console.log(await rpc.notes.list())'
zen exec ./script.ts
```

Inside the script:

```typescript
import { rpc, events } from "@zenbujs/cli/exec"

const notes = await rpc.notes.list()
events.notes.noteAdded.subscribe(payload => console.log(payload))
```

The script connects over the same websocket the renderer uses. Long-running scripts can `await new Promise(...)` to stay alive.

## `zen profile`

Profile a running kernel:

```bash
zen profile --duration 5000 --out cpu.prof
zen profile heap --out heap.heapsnapshot
```

Outputs are loadable in Chrome devtools.

## `zen plugin`

```bash
zen plugin list                       # all plugins, with enabled state
zen plugin info <name>                # manifest fields
zen plugin toggle <name> <on|off>     # enable/disable
zen plugin remove <name>              # remove from config.json (does not delete files)
```

## `zen runtime`

Manage the framework's cached Electron runtimes:

```bash
zen runtime install <version>
zen runtime list
zen runtime remove <version>
```

Used when the framework wants a different Electron version than your project's; rare.

## `zen build:source`

Run the TS-to-TS source build defined in `zenbu.build.ts`. See [Build & distribution](/build/overview).

## `zen build:desktop`

Bundle the staged source into a runnable binary. Calls `build:source` first, then hands off to the configured adapter.

## `zen publish:source [init|push]`

Push the staged source to the configured mirror repo. Live-source mode only.

## `zen publish:desktop`

Upload the binary to a release host (currently GitHub Releases).

```bash
zen publish:desktop --host github --tag v0.4.2 --token "$GITHUB_TOKEN"
```
