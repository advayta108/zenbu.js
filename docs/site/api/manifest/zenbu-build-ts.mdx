# `zenbu.build.ts`

The build configuration file for `nr build:source` and `nr build:desktop`. Lives at the plugin root next to `zenbu.plugin.json`.

```typescript
import { defineBuildConfig } from "@zenbujs/cli/build"
import { liveSource } from "@zenbujs/cli/adapters/live-source"

export default defineBuildConfig({
  source: ".",
  out: ".zenbu/source",
  include: [/* ... */],
  ignore: [/* ... */],
  transforms: [/* ... */],
  adapter: liveSource({/* ... */}),
  hooks: [/* ... */],
})
```

## Field reference

See [Define build config](/build/define-build-config) for full semantics. The TypeScript types are in `@zenbujs/cli/build`:

```typescript
interface BuildConfig {
  source?: string                       // default "."
  out?: string                          // default ".zenbu/source"
  include: string[]                     // required
  ignore?: string[]
  transforms?: Transform[]
  adapter?: Adapter                     // default liveSource()
  hooks?: Hook[]
}
```

## Adapter selection

The single most important field is `adapter:`. It selects the distribution model:

```typescript
import { liveSource } from "@zenbujs/cli/adapters/live-source"
import { frozenSource } from "@zenbujs/cli/adapters/frozen-source"

// Default — ships TS source, supports live updates over git.
adapter: liveSource({ mirror: { target: "owner/release-repo" } })

// Frozen — pre-compiled JS, asar-bundled, no toolchain.
adapter: frozenSource()
```

Comment the line out and the framework injects `liveSource()` with default options.

## Reserved fields

The framework reserves the following top-level fields for future use:

- `bundle` (legacy — use `adapter.bundle` if exposed by your adapter)
- `mirror` (legacy — use `adapter: liveSource({ mirror: {...} })`)
- `presets`

## Validation

`defineBuildConfig` is type-checked. The CLI also performs runtime validation: `nr build:source` exits with a clear error if `include` is empty, `source` doesn't exist, or any `transform` isn't a function.
