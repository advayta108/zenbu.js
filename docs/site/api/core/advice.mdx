# `@zenbujs/core/advice`

```typescript
import { advise, inject, type AdviceSpec } from "@zenbujs/core/advice"
```

Two functions, one type. That's the entire public surface.

## `inject(spec, importMeta?)`

Register a content script.

```typescript
function inject(
  spec: { view: string; module: string },
  importMeta?: ImportMeta,
): () => void
```

| Field | Required | Meaning |
| --- | --- | --- |
| `view` | yes | View scope id, or `"*"` for all views. |
| `module` | yes | Path to the content-script module. Relative to the plugin root if `importMeta` is provided; otherwise must be absolute. |

Returns an unregister function. Wrap with `useDisposable` to tie it to your service's lifecycle.

## `advise(spec, importMeta?)`

Register function advice.

```typescript
function advise(spec: AdviceSpec, importMeta?: ImportMeta): () => void

type AdviceSpec = {
  view: string                                  // scope id, or "*"
  target: { module: string; export: string }    // what to advise
  kind: "replace" | "before" | "after" | "around"
  handler: { module: string; export: string }   // your handler
}
```

The `target.module` is resolved against the *targeted view's* renderer source root.
The `handler.module` is resolved against *your* plugin root (when `importMeta` is provided).

Returns an unregister function.

## What's *not* public

- `registerAdvice` / `registerContentScript` — the **previous** names of `advise` / `inject`. Re-exported with deprecation warnings; will be removed in the first major.
- `getAdvice`, `getContentScripts`, `getAllScopes`, `getAllAdviceScopes`, `getAllContentScriptPaths` — internal accumulators read by the prelude generator. **Not public.** If you need to introspect, query the [`runtime` service](/runtime/runtime-control) or use `nr zen advice list <scope>`.
- `ViewAdviceEntry` — the resolved entry shape. Internal; the public-facing type is `AdviceSpec`.

## Renamed identifiers

The current code base uses these names; they're being renamed for consistency. Both work for now; the new names are canonical.

| Old | New |
| --- | --- |
| `registerAdvice(scope, entry, meta)` | `advise(spec, meta)` |
| `registerContentScript(scope, modulePath, meta)` | `inject(spec, meta)` |
| `entry.scope` | `spec.view` |
| `entry.type` | `spec.kind` |
| `entry.moduleId` | `spec.target.module` |
| `entry.name` | `spec.target.export` |
| `entry.modulePath` | `spec.handler.module` |
| `entry.exportName` | `spec.handler.export` |

The shape change is mechanical. A codemod will be available when the migration ships.
