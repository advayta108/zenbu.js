# Views API

The views system has two service-level APIs (`Views` and `Window`) and one Vite helper (`defineZenbuViewConfig`). All are imported from `@zenbujs/core/services` or `@zenbujs/core/vite`.

## `Views` service

Imported from `@zenbujs/core/services`:

```typescript
import { Views } from "@zenbujs/core/services"
```

### `register(options)`

```typescript
register(options: {
  id: string
  root: string
  viteConfig?: string | false
  panel?: PanelOptions
}): () => void
```

Registers a view. Returns an unregister function. Pair with `useDisposable("...", () => views.register({...}))` so it tears down on reload.

### `registerAlias(options)`

```typescript
registerAlias(options: {
  id: string
  reloaderId: string
  pathPrefix?: string
  panel?: PanelOptions
}): () => void
```

Registers a view that reuses another view's Vite server. `reloaderId` is the `id` of the parent view.

### `unregister(id)`

```typescript
unregister(id: string): Promise<void>
```

Unregister a previously registered view. You usually use the cleanup returned by `register` instead.

### `get(id)`

```typescript
get(id: string): { url: string; port: number } | undefined
```

Look up a registered view's URL. Useful for debug tooling.

## `MAIN_WINDOW_ID`

```typescript
const MAIN_WINDOW_ID: string
```

The string used as the default `windowId` for `openView` and other window APIs. Currently `"main"`. Re-exported so you don't hard-code it.

## `defineZenbuViewConfig(extra?)`

```typescript
import { defineZenbuViewConfig } from "@zenbujs/core/vite"

export default defineZenbuViewConfig({
  // any standard Vite UserConfig fields
  plugins: [tailwind()],
})
```

A helper that returns a Vite `UserConfig` with React + the framework's mandatory plugins (advice prelude, content-script discovery, theme stylesheet, framework resolve). Plugins use this to avoid installing Vite tooling in their own dependency tree.

## Types

```typescript
type PanelOptions = {
  sidebar?: boolean
  bottomPanel?: boolean
  label?: string
  kind?: string
}

type ViewEntry = {
  scope: string                // matches the registered `id`
  url: string
  port: number
  ownsServer: boolean          // true for register(), false for registerAlias()
  meta?: PanelOptions
}
```

`ViewEntry` is what's stored in `root.plugin.core.lastKnownViewRegistry`. Read it from a renderer via `useDb`.

## What's *not* public

- `BaseWindowService` — internal Electron `BaseWindow` management. Use `Window` instead.
- `RendererHostService` — internal mapping of `manifest.uiEntrypoint` to a registered view.
- `ReloaderService` — the Vite-server pool. Use `Views` to register/get; don't touch the reloader directly.
- `LocalFileProtocolService` — internal `local://` resource serving. May be exposed in a future minor for plugins that want to ship custom file protocols.

## Removed / deprecated

- `WindowService` (lowercase service class export) — renamed to `Window`. The old name still re-exports for one minor.
- `ViewRegistryService` — renamed to `Views`. Same.
