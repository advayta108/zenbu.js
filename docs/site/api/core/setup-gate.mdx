# `@zenbujs/core/setup-gate`

```typescript
import { setupGate, bootstrapEnv } from "@zenbujs/core/setup-gate"
```

Two exports. Both are advanced — you only call them when [customizing boot](/boot/customizing-boot).

## `setupGate()`

```typescript
function setupGate(): Promise<void>
```

The framework's main-process entrypoint. Performs all the work documented in [Boot overview](/boot/overview). Resolves when the runtime is idle (services ready, no reload pending).

Throws if Electron isn't available (i.e. you're trying to run this in plain Node).

### When to call

Only when overriding `package.json#main`. The default `main` already calls this; calling it again is a no-op.

## `bootstrapEnv()`

```typescript
function bootstrapEnv(): void
```

Synchronously populates `process.env` with framework paths:

- `PATH` is augmented with the toolchain dir.
- `ZENBU_INTERNAL_DIR` is set to the framework cache root.
- `ZENBU_APPS_DIR` is set if not already.

`setupGate()` calls this internally. You only call it manually if you have code that needs the env populated *before* `setupGate()` runs (e.g. a child-process spawn that needs `PATH` to find `pnpm`).

## Env vars the framework reads

| Variable | Effect |
| --- | --- |
| `ZENBU_CONFIG_PATH` | Override the location of `config.json`. |
| `ZENBU_APPS_DIR` | Override the parent of per-app dirs (default: `~/.zenbu/apps`). |
| `ZENBU_INTERNAL_DIR` | Override the framework cache dir (default: `~/.zenbu/.internal`). |
| `ZENBU_VERBOSE` | Set to `1` to print verbose boot logs and slow-spans. |
| `ZENBU_ADVICE_ROOT` | Internal — set by `setupGate()` to the project root, used by the advice transform. |

Anything else under `ZENBU_*` is reserved for future use; don't squat on the namespace.

## What's *not* public

- `defaultServices()` from `@zenbujs/core/services/default` — internal. Triggered by `setupGate()`.
- The setup-gate's internal helpers (`resolveProjectRoot`, `loadElectronApp`, etc.). They're stable enough that we may expose them later, but for now they're internal.
