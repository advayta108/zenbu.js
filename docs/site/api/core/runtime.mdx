# `@zenbujs/core/runtime`

The runtime exposes seven public symbols. Anything not listed here is internal, even if it's accidentally exported by the package today.

```typescript
import {
  Service,
  optional,
  runtime,
  type CleanupReason,
  type ServiceRuntime,
} from "@zenbujs/core/runtime"
```

---

## `Service`

Abstract base class for every service.

```typescript
abstract class Service {
  static key: string
  static deps: Record<string, ServiceRef | OptionalServiceRef>

  ctx: { /* injected by runtime, typed via Service.with(...) */ }

  /**
   * Lifecycle hook called once after all required deps are ready.
   * Awaited by the runtime; thrown errors put the service in `failed`.
   */
  start(): void | Promise<void>

  /**
   * Register a disposable resource. The teardown function (if returned) runs
   * before the next reload and on shutdown.
   *
   * @param key — identifier within this service. Re-registering with the same
   *   key tears down the previous instance first; useful for HMR.
   * @param fn — the setup function. Returns a teardown function or void.
   */
  useDisposable(
    key: string,
    fn: () => void | DisposeFn | Promise<void | DisposeFn>,
  ): void
}

type DisposeFn = (reason: CleanupReason) => void | Promise<void>
type CleanupReason = "reload" | "shutdown"
```

### Example

```typescript
import { Service, runtime } from "@zenbujs/core/runtime"

export class TickerService extends Service {
  static key = "ticker"

  async start() {
    this.useDisposable("interval", () => {
      const id = setInterval(() => console.log("tick"), 1000)
      return () => clearInterval(id)
    })
  }
}

runtime.register(TickerService, import.meta)
```

### Notes

- `static key` is required and must be globally unique within the process.
- A service that needs **no** deps can extend `Service` directly. Otherwise extend `Service.with({...})`.
- `start()` is optional. Define it only if you have async initialization.

## `Service.with({...})`

Static helper that returns a typed base class with `static deps` pre-set and `this.ctx` typed.

```typescript
Service.with<TDeps extends Record<string, ServiceRef | OptionalServiceRef>>(
  deps: TDeps,
): typeof Service & { new (): Service & { ctx: ResolvedDeps<TDeps> } }
```

### Example

```typescript
export class RegistryWatcher extends Service.with({
  db: DbService,                           // required
  http: HttpService,                       // required
  agents: optional(AgentManagerService),   // optional
}) {
  static key = "registry-watcher"

  async start() {
    this.ctx.db                            // DbService
    this.ctx.http                          // HttpService
    this.ctx.agents                        // AgentManagerService | undefined
  }
}
```

### Notes

- The dep map can mix typed class refs and string keys. String keys produce `unknown` types; class refs produce `InstanceType<typeof Class>` (or `| undefined` when wrapped in `optional()`).
- A service may also use `static deps = {...}` directly without `Service.with(...)`. Doing so trades the auto-typing of `this.ctx` for slightly more explicit class layout. Prefer `Service.with(...)`.

## `optional(ref)`

Wraps a service reference to make it optional.

```typescript
function optional<R extends ServiceRef>(ref: R): OptionalServiceRef<R>
```

- `optional(SomeService)` → `this.ctx.someService: SomeService | undefined`
- `optional("some-key")` → `this.ctx.someKey: unknown | undefined`

If the optional dep becomes available (or unavailable) at runtime, the service is reloaded so the change in `ctx` is observed.

## `runtime`

The process-singleton `ServiceRuntime`. Imported as a named export:

```typescript
import { runtime } from "@zenbujs/core/runtime"
```

### `runtime.register(ServiceClass, importMeta?)`

Registers a service class. The second argument should always be `import.meta`.

```typescript
runtime.register(MyService, import.meta)
```

If a class with the same `static key` is already registered, the old instance is torn down and replaced. This is what makes hot reload work.

### `runtime.get(ServiceClass)`

Look up a started service instance by its class.

```typescript
const counter = runtime.get(CounterService)
```

Throws if the service isn't registered or isn't `ready`. **Do not call this from a service that should depend on the looked-up one** — declare it as a dep instead. See [Dependencies — anti-pattern](/runtime/dependencies#anti-pattern%3A-dynamic-deps).

### `runtime.shutdown()`

Tear down every registered service in reverse-topological order.

```typescript
await runtime.shutdown()
```

Called automatically by `setupGate()` on `before-quit`. You don't normally call this yourself.

### `runtime.reloadAll()`

Mark every service dirty and run reconcile.

```typescript
await runtime.reloadAll()
```

Used by the [`runtime` service's RPC](/runtime/runtime-control) to power the "reload all" command.

### `runtime.onReconciled(cb)`

Subscribe to reconcile completions.

```typescript
const off = runtime.onReconciled(changedKeys => { /* ... */ })
off()                                    // unsubscribe
```

Fires after every reconcile pass — boot, hot-reload, optional-dep flips. The argument is the list of service keys that just changed state (started, stopped, failed, recovered, etc.).

## Removed / not public

The following are **not part of the public API**, even though they may be exported from the package today. They are slated for either removal or migration to a separate `@zenbujs/core/internals` subpath:

- `serviceWithDeps()` — replaced by `Service.with(...)`. Will be removed.
- `runtime.buildRouter()` — internal to RpcService.
- `runtime.getSlot()`, `runtime.getAllKeys()` — visible from the [`runtime` service](/runtime/runtime-control), but as method calls, not direct module access. Will move to a `@zenbujs/core/runtime/inspect` subpath.
- `Service.prototype.trace()`, `Service.prototype.traceSync()` — observability internals. Use the [boot bus](/runtime/runtime-control) for telemetry.
- `ServiceRuntime` class export — only the singleton `runtime` is public; constructing a second runtime is unsupported.

If you depend on any of these today, see [Migration recipe](/recipes/migrate-from-effect) for the bridge plan.
