# Add an RPC procedure

The shortest path from a service method to a callable RPC procedure.

## 1. Add an `async` method to your service

```typescript
// src/main/services/notes.ts
export class NotesService extends Service.with({ db: DbService }) {
  static key = "notes"

  async pin(noteId: string) {
    await this.ctx.db.update(r => {
      const note = r.plugin["notes-plugin"].notes.find(n => n.id === noteId)
      if (note) note.pinned = true
    })
  }
}
```

If the method is `async` and its name doesn't start with `_`, it's automatically RPC-callable. **There is no separate registration step.**

## 2. Save the file

The runtime hot-reloads the service. The new RPC method is now visible to renderers (the websocket reconnects so the renderer's typed proxy picks up the change).

## 3. Call it from the renderer

```tsx
import { useRpc } from "@zenbujs/core/react"

function PinButton({ noteId }: { noteId: string }) {
  const rpc = useRpc()
  return <button onClick={() => rpc.notes.pin(noteId)}>Pin</button>
}
```

`rpc.notes.pin` is fully typed: same arguments and return type as the server method.

## Things to know

- Throw to reject. The renderer's promise rejects with an `Error` matching the message.
- Optimistic updates flow through `db.update(...)` automatically. If your method modifies state, the renderer sees the update before RPC resolves.
- Non-`async` methods are not exposed. If you have a sync method you also want over RPC, write a thin async wrapper.
- Methods starting with `_` are not exposed. Use this to mark internal helpers.

## Hiding a method

Make it `private` or prefix with `_`:

```typescript
async pin(noteId: string) { /* exposed */ }
async _internalPin(noteId: string) { /* not exposed */ }
private async pinHelper() { /* not exposed */ }
```
